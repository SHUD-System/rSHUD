# Project Structure

## Directory Organization

```
rSHUD/
├── R/                      # R source code (main package functions)
├── src/                    # C++ source code (Rcpp integration)
├── man/                    # Documentation files (auto-generated by roxygen2)
├── data/                   # Example datasets (.rda files)
├── demo/                   # Example scripts and workflows
├── experiments/            # Experimental/development code
├── nosync/                 # Local development files (not synced)
├── .Rproj.user/           # RStudio project files (gitignored)
├── DESCRIPTION             # Package metadata and dependencies
├── NAMESPACE               # Exported functions (auto-generated)
├── LICENSE                 # MIT license
├── README.md               # Main documentation
└── .gitignore             # Git ignore rules
```

## R/ Directory - Functional Organization

Functions are organized by domain/purpose:

### Core Model Building
- `autoBuildModel.R` - Main automated model building workflow
- `MeshDomain.R` - Mesh domain creation and management
- `ModelConfigure.R` - Model configuration (parameters, calibration)
- `ModelClasses.R` - S4 class definitions (SHUD.MESH, SHUD.RIVER)
- `ModelInfo.R` - Model information utilities
- `Project.R` - Project management functions

### Input/Output
- `readinput.R` - Read SHUD input files (.mesh, .riv, .att, .ic, .para, .calib, etc.)
- `readout.R` - Read SHUD output files
- `readTSD.R` - Read time series data
- `writeInput.R` - Write SHUD input files
- `NetCDF.R` - NetCDF file handling

### GIS Processing
- `GIS_delineation.R` - Watershed delineation
- `GIS_RiverProcess.R` - River network processing
- `GIS_SimplifyPolylines.R` - Polyline simplification
- `GIS_SimplifybyLen.R` - Length-based simplification
- `GIS_Projection.R` - Coordinate projection utilities
- `Func_GIS.R` - General GIS utility functions
- `trianglulate.R` - Mesh triangulation

### Hydrological Calculations
- `PET.R` - Potential evapotranspiration (Penman-Monteith)
- `WaterBalance.R` - Water balance analysis
- `Hydro_obs.R` - Hydrological observations processing
- `ModelBC.R` - Boundary conditions

### Data Processing
- `River.R` - River network data structures and operations
- `Sinks.R` - Sink detection and handling
- `DataFilter.R` - Data filtering utilities
- `Func_Get.R` - Data extraction functions
- `Func_Misc.R` - Miscellaneous utilities
- `Func_PTF.R` - Pedotransfer functions (soil/geology parameters)
- `Func_landcover.R` - Land cover processing

### Spatial Data
- `NLCD.R` - National Land Cover Database utilities
- `plotMap.R` - Spatial mapping functions
- `plot.R` - General plotting functions

### Package Infrastructure
- `SHUD_Env.R` - Package environment and settings
- `RcppExports.R` - Auto-generated Rcpp exports

## Key Conventions

### File Naming
- R source files use PascalCase or descriptive names (e.g., `GIS_delineation.R`, `autoBuildModel.R`)
- Functions within files typically match or relate to the filename
- C++ files use camelCase (e.g., `polygonArea.cpp`)

### Class Definitions
- S4 classes defined in `ModelClasses.R`
- Main classes: `SHUD.MESH`, `SHUD.RIVER`, `Untructure Domain`
- Use `@` for slot access in S4 objects

### Function Naming
- Use dot notation for related functions: `read.df()`, `write.df()`, `read.tsd()`, `write.tsd()`
- Prefix conventions:
  - `read*` - Input functions
  - `write*` - Output functions
  - `sp.*` - Spatial operations
  - `shud.*` - SHUD-specific constructors/utilities
  - `get*` - Data extraction
  - `PTF.*` - Pedotransfer functions

### Documentation
- All exported functions documented with roxygen2 comments
- Documentation includes: title, description, parameters (`@param`), return value (`@return`), examples (`@examples`), exports (`@export`)
- Man pages auto-generated - never edit `.Rd` files directly

### Output Organization
Model outputs typically organized as:
```
output/
├── fig/          # Generated figures/plots
├── gis/          # Shapefiles and spatial outputs
└── [model files] # SHUD input files (.mesh, .riv, etc.)
```
